## Decision Rule

- `KEEP`: required by the one-PC Control Desk or the connected access chain.
- `KEEP-GAP`: required, but the current top-level screen is only a shell or does not yet expose the complete use case.
- `DEFER`: working or partially working capability that is outside the Lean V1 acceptance path. Keep it compiled and tested, but hide it from routine operator navigation.
- `REMOVE-CANDIDATE`: excluded and apparently disconnected work that still requires a caller, migration, persistence, and test impact proof before deletion.

## Client 360 Subfeature Boundary

| Tab / action family | Decision | Reason |
| --- | --- | --- |
| Overview, client identity/status, primary contact, support notes | `KEEP` | Required client record and operator context. |
| Simple active contract and deployment profile | `KEEP` | Supplies commercial and installation authority for desired access. |
| Invoice draft/issue and payment record/approval status | `KEEP` | Required commercial evidence before access approval. |
| Voucher/source-document proof generated by invoice/payment workflows | `KEEP` | Minimum balanced accounting evidence; no manual journal dependency. |
| Entitlement/access revision issuance | `KEEP` | Creates immutable desired access. |
| Outbox publication, installation status, heartbeat, delivery/observed comparison | `KEEP` | Required Control Desk -> Control Cloud -> Local Server -> Control Desk loop. |
| Client Portal invitations, resend/revoke, portal payment claims, and portal MFA | `DEFER` | The Client Portal and SMTP/Brevo are explicitly outside Lean V1. |
| Manual journals, period close/reopen, broad COA maintenance, P&L/balance sheet/trial-balance workspace | `DEFER` | Advanced accounting is not a Lean V1 release dependency. |
| App activation/support commands beyond setup, entitlement delivery, heartbeat, and essential recovery | `DEFER` | Optional operational breadth must not enlarge the first reliable chain. |

## Concrete Navigation Tightening

The next UI change is deliberately small:

1. keep Command Center, Client 360, Client Money, Cloud & Installation, Setup, and Access & Security visible;
2. hide Voucher Register, broad Reports & Audit, and Admin Desk from primary navigation;
3. prevent hashes for hidden sections from reopening them through routine navigation;
4. remove the Setup-to-Admin-Desk shortcut;
5. label required placeholder screens honestly until their minimal pages are wired;
6. hide portal-only actions inside Client 360 without deleting their backend code.

## Findings That Change the Plan

- The repository does not need a restart. The working Client 360 already calls the essential client, contract, billing, payment, entitlement, and cloud APIs.
- Several polished top-level areas are shells, not completed workspaces. They must not be counted as delivered merely because they are visible.
- Accounting is not simply dead code. Automatic journals and client financial evidence are coupled to the required commercial chain, while the broad operator workbench is deferrable.
- Survey/FAS is the first removal candidate, but deletion remains a separate reversible task after dependency and persisted-data proof.
